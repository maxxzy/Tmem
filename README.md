# 简单方案设计

## agent memory为什么需要主题

1.向量检索依赖语义相似度，可能无法区分主题。比如查询"我上次面试准备了什么"时，可能召回"我准备了过年的年货"——两条记忆中"准备"语义相近，但主题不同。

2.用户查询隐含着主题，比如"我为什么想换工作"→"工作"。如果能识别query的相关主题，搜索空间就能从全部记忆缩小到该主题下的n条，过滤掉噪声记忆，提升精确率。   ***？我认为主题过滤的核心作用是过滤掉其他噪声记忆以提高精确率，而不是缩小搜索范围***

3.比如问题“我为什么想换工作”包含多个关联主题“工作”、“工作压力”、“生活质量”等等，构建主题间的关系索引可以为memory检索提供导航路径，如果仅仅依赖向量相似度，难组织好的多跳导航链

## agent memory中主题问题的难点

1.主题边界问题：真实对话中主题切换是渐进的变化的，难以准确划定主题边界

2.主题粒度问题：比如"工作"与"暑期实习"是不同粒度的主题，需要组织层次化的结构来管理主题

3.query的主题模糊：用户查询短而且模糊，主题不明确，路由到错误的主题会导致结果完全偏离

## 整体解决方案

1.主题感知的记忆抽取：对话->主题分段->主题标签->带主题标签的记忆

2.主题层次索引构建：主题集->多粒度聚类->主题DAG + 主题关联图

3.主题路由的记忆检索：查询->主题路由->主题内检索->跨主题扩展

## 主题感知的记忆抽取

**总体思路总结：**针对现有记忆抽取方法忽略主题连续性、容易将完整语义单元切碎的问题，本课题拟提出一种主题感知的记忆抽取机制，以主题分段替代固定窗口切分，将相邻对话的语义变化与显式转折信号结合起来识别主题边界，并借助轻量LLM为主题段生成多主题标签、关键词和结构化记忆。通过让每条记忆在抽取阶段即绑定一个或多个高置信主题，该机制能够从源头减少主题漂移和标签缺失问题，为后续的主题组织、关联建图与检索路由提供一致、可扩展的输入表示。

**核心：**识别主题边界，使每条抽取出的记忆都携带准确的主题标签

mem0逐轮抽取记忆，忽略了主题。LightMem窗口摘要，主题可能被切断

改进方案：用主题边界检测代替固定窗口，确保每个抽取单元是一个完整的主题段

**具体方法**：

​	1.对相邻对话计算语义相似度，唱过阈值时设为候选的主题边界，同时考虑对话中的一些转折词（比如“另外”），将对话分割成多个主题段

​	2.对每个主体段利用llm（轻量的）生成主题标签（1个或者多个，短语）和关键词（每个标签对应的关键词集合），将每个主题标签与已有主题集合做比较，若相似度超过阈值，则归入已有的主题。最终每个主题段都关联一个主题集合

​        3.主题段内，使用llm生成结构化的记忆，并要求生成记忆时从该主题段关联的主题集合（这样不会有多余的新主题出现）中选取涉及到的全部主题，作为该记忆的主题标签

***为什么一个记忆要多主题而不是单主题***

单主题情况下，如果检索时主题路由出错，就会丢失这个记忆，如果是多主题则可以增加容错

真实对话的语境下本来就是一句话包含多个主题，更符合真实情况

一个记忆有多个主题，在后续构建主题关联图的时候，形成了一种显示的主题之间的关系，比纯靠语义去关联更好

同时，因为主题段是由相邻对话计算语义相似度来判定的，最终多个主题还是单个主题，对主题段的判定没有影响

## 主题层次索引构建

**总体思路总结：**针对记忆主题同时具有层次性、跨层关联性和动态演化性的特点，本课题拟构建一种由主题层次DAG与主题关联图共同组成的主题层次索引体系，对抽取得到的主题进行统一组织、压缩表示与增量维护。该索引体系一方面利用层次结构支持粗粒度聚合和细粒度定位，另一方面利用关联结构支持跨主题联想与推理扩展，从而在提升主题过滤精度的同时，为复杂查询下的高效记忆检索提供结构化支撑。

**核心：** 根据主题过滤记忆，使检索时能同时支持粗粒度主题的聚合查询和细粒度主题的定位查询（主题DAG），同时能支持跨主题的推理查询（主题关联图）

如果不构建层次化的主题索引，面对查询”我学过什么编程”，可能无法一次性聚合多个相关主题比如”java”、”python”

相关但语义不相似的主题比如”工作压力”与”失眠”之间的关系不明显

### 二、构建主题层次DAG（有向无环图）

**总体思路总结：**针对主题天然具有多粒度层次且可能同时归属于多个上位类别的问题，本课题拟提出一种主题层次DAG索引结构对抽取主题进行高效存储与组织。该结构通过融合主题标签、关键词、主题摘要等多视角信息构建主题表示，先以多粒度聚类形成初始层次，再通过多父边补充和增量更新机制扩展为可动态演化的DAG。通过构建兼顾层次性与多归属性的主题索引，本方法能够有效解决严格树结构表达受限、增量维护困难等问题，并保证主题感知记忆检索中的聚合查询、精确定位和动态扩展效率。

***为什么用DAG而不是树？***

原方案使用HDBSCAN聚类构建严格的主题树，存在两个问题：

1. **多归属限制**：真实世界中一个主题可以属于多个父主题。例如”强化学习”既是”机器学习”的子主题，也属于”决策科学”。严格树结构要求唯一父节点，会丢失这种天然的多归属关系
2. **静态固化**：HDBSCAN一次性聚类产生的层次在记忆增量增长时难以动态调整，需要反复重聚类

> **相关工作**：Topic-DAG模型（2023-2024）提出用DAG替代树来建模主题层次，其中每个主题允许多个父节点。TaxoEnrich（Jiang et al., 2023）和TEMP（2024）进一步研究了如何在已有DAG分类体系中为新主题找到多个合适的父节点。HyHTM（Dufraisse et al., 2023, ACL Findings）提出在双曲空间（Poincaré球模型）中嵌入主题层次，利用双曲空间指数增长的容积特性天然匹配主题树的指数分支结构。

**具体方法——主题DAG的构建流程：**

#### 步骤1：主题向量表征

每个主题节点 $t_i$ 由以下信息表征：
- **标签嵌入向量** $\mathbf{e}_i$：对主题标签短语用sentence embedding模型编码
- **关键词集合** $K_i$：该主题的关键词集
- **记忆集合** $M_i$：归属于该主题的记忆id集合
- **主题摘要** $S_i$：由LLM对该主题下记忆集合生成的1-3句概括性描述（参考Generative Agents（Park et al., 2023）的reflection机制，将隐式的反思层级显式化为DAG节点属性）
- **摘要嵌入** $\mathbf{e}^s_i$：对摘要文本编码的向量

> **摘要的价值**：查询路由时，可以用query与主题摘要的相似度做匹配，比仅靠短主题标签嵌入更准确——标签通常只有2-4个词，信息量有限；摘要携带了该主题下记忆的核心语义。此外，上层虚拟节点的摘要可以作为"主题目录"常驻LLM上下文，为agent提供全局记忆脉络（参考MemGPT（Packer et al., 2023）的core memory思路）。
>
> **摘要更新策略**：每当主题下新增5条记忆，或主题发生合并/分裂时，由LLM重新生成摘要并更新摘要嵌入。

计算两个主题之间的综合相似度：

$$sim(t_i, t_j) = \alpha_1 \cdot cos(\mathbf{e}_i, \mathbf{e}_j) + \alpha_2 \cdot cos(\mathbf{e}^s_i, \mathbf{e}^s_j) + \alpha_3 \cdot Jaccard(K_i, K_j)$$

其中 $\alpha_1 + \alpha_2 + \alpha_3 = 1$ 为平衡系数（推荐 $\alpha_1=0.4, \alpha_2=0.3, \alpha_3=0.3$），$cos$ 为余弦相似度，$Jaccard$ 为关键词集合的Jaccard相似系数：

$$Jaccard(K_i, K_j) = \frac{|K_i \cap K_j|}{|K_i \cup K_j|}$$

> 引入摘要嵌入相似度后，两个标签不同但描述相近语义的主题（如"加班"和"工时过长"）可通过摘要层面的语义匹配被正确识别为高相似。

#### 步骤2：初始层次发现——多粒度聚类

对全部主题使用 **层次凝聚聚类（HAC）**，以 $1 - sim(t_i, t_j)$ 为距离度量，采用average-linkage：

1. 设定多个距离阈值 $\theta_1 > \theta_2 > ... > \theta_L$（例如L=3层，$\theta = [0.7, 0.5, 0.3]$），在每个阈值处切割树状图，得到从粗到细的聚类结果
2. 每一层的聚类簇形成该层的**虚拟主题节点**（由LLM为每个簇命名），叶子节点为原始具体主题

初始结果形如：

```
             ROOT（所有主题）
           /        |        \
    技术学习      工作生活     健康     ──── 第1层虚拟节点（粗粒度簇）
    /    \        /   \        |
 编程   数据库  工作压力 职业规划  睡眠   ──── 第2层虚拟节点（中粒度簇）
 / \     |       |      |      |
Py Java MySQL  加班   面试准备  失眠   ──── 第3层叶子节点（具体主题）
```

#### 步骤3：从树到DAG——添加多父边

在初始层次树基础上，识别需要多归属的主题并添加跨分支的父子边：

**方法（a）：基于相似度的候选父节点发现**

对每个主题 $t_i$，在非当前父节点的上层节点中，找到满足以下条件的备选父节点 $t_p$：

$$sim(t_i, t_p) > \beta \quad \text{且} \quad depth(t_p) < depth(t_i)$$

其中 $\beta$ 为多归属阈值（推荐 $\beta = 0.6$）

**方法（b）：LLM辅助判断**

将候选的 $(t_i, t_p)$ 对提交给LLM，利用prompt判断”$t_i$ 是否可以被视为 $t_p$ 的子主题”，只保留LLM确认的边。

> 这里使用LLM作为最终裁判，而非仅依赖向量相似度，是因为”子主题”关系是一种语义上的is-a/part-of关系，向量相似度高不一定代表层次隶属关系。参考LLMs4OL（Babaei Giglou et al., 2023）的研究，GPT级别模型在taxonomy discovery任务上表现良好。

**添加多父边后的DAG示例：**

```
               ROOT
           /     |     \
    技术学习   工作生活   健康
    /  \   \    / \       |
 编程  DB  数据分析 工作压力 睡眠
 / \        \        |
Py Java    pandas   加班
```
（”数据分析”同时属于”技术学习”和”工作生活”）

#### 步骤4：双曲空间嵌入（可选增强）

> 参考HyHTM（Dufraisse et al., 2023），将DAG节点嵌入到Poincaré球模型中

将每个主题节点映射到Poincaré球 $\mathbb{B}^d = \{x \in \mathbb{R}^d : \|x\| < 1\}$，使用双曲距离：

$$d_{\mathcal{P}}(\mathbf{u}, \mathbf{v}) = \text{arccosh}\left(1 + 2\frac{\|\mathbf{u} - \mathbf{v}\|^2}{(1-\|\mathbf{u}\|^2)(1-\|\mathbf{v}\|^2)}\right)$$

**层次深度由向量范数编码**：根节点靠近原点（$\|\mathbf{z}\| \approx 0$），叶子节点靠近边界（$\|\mathbf{z}\| \approx 1$）

**父子关系由蕴涵锥（Entailment Cone）建模**：子节点的嵌入落在父节点所发出的锥体内

**优势**：
- 双曲空间容积随半径指数增长，天然匹配主题层次的指数分支
- 可以通过向量范数快速判断主题的粒度层级（粗/细）
- 通过锥体成员检测可以O(1)查询父节点
- 为后续查询路由提供连续的层次空间，支持在层次结构中做最近邻搜索

**训练目标**：对DAG中的边 $(parent, child)$ 最小化以下损失：

$$\mathcal{L} = \sum_{(p,c) \in E_{DAG}} \max(0, d_{\mathcal{P}}(\mathbf{z}_p, \mathbf{z}_c) - d_{\mathcal{P}}(\mathbf{z}_p, \mathbf{z}_{c'}) + \gamma)$$

其中 $c'$ 为负样本（非子节点），$\gamma$ 为margin

#### 步骤5：动态更新策略

当新主题 $t_{new}$ 产生时（从”主题感知的记忆抽取”阶段传入的新主题标签无法归入已有集合）：

1. **计算候选父节点**：在现有DAG中，从根节点开始贪心下降，每一层选择与 $t_{new}$ 相似度最高的节点，直到 $sim(t_{new}, t_{child}) < \beta$ 或到达叶子
2. **LLM确认**：将候选位置提交LLM确认父子关系
3. **插入DAG**：添加节点和确认的父子边
4. （若使用双曲嵌入）在Poincaré球中初始化新节点位置为其父节点方向上范数更大的位置，进行局部fine-tune

**主题漂移检测**：当一个主题下新增记忆的关键词分布与原有分布差异过大时（用Jensen-Shannon散度度量）：

$$JSD(P_{old} \| P_{new}) = \frac{1}{2}KL(P_{old}\|M) + \frac{1}{2}KL(P_{new}\|M), \quad M = \frac{P_{old}+P_{new}}{2}$$

当 $JSD > \delta$（推荐 $\delta=0.3$）时，触发主题分裂：将该主题拆分为两个子主题，由LLM重新命名

**主题合并**：定期检查，当两个兄弟主题的 $sim(t_i, t_j) > \mu$（推荐 $\mu=0.85$）且各自记忆数量均低于阈值时，合并为一个主题

> **相关工作**：BERTopic（Grootendorst, 2022及后续temporal扩展）提供了主题随时间演变的跟踪机制。TaxoComplete（Jiang et al., 2023）研究了在已有taxonomy中动态插入新概念。动态主题模型（Neural DTM, 2023-2024）将主题在时间步之间的演变建模为：$\alpha_t = \alpha_{t-1} + \epsilon_t, \, \epsilon_t \sim \mathcal{N}(0, \sigma^2 I)$

### 三、构建主题关联图

**总体思路总结：**针对仅依赖层次结构难以刻画主题之间因果、共现和时序联系的问题，本课题拟进一步构建一个独立于主题DAG的主题关联图，用于显式表示跨主题的非层次关系。该关联图综合多主题记忆共现、LLM关系判断以及时序共现等多源信号生成带权有向边，并通过剪枝、衰减与增量更新保持图结构的稀疏性与时效性。通过构建主题间的联想通路，系统能够在语义不完全相近但实际高度相关的主题之间实现有效跳转，从而提升复杂问题场景下的跨主题召回能力和多跳推理能力。

主题关联图是一个**加权有向图** $G = (V, E, w)$，其中 $V$ 为全部主题节点，$E$ 为关联边，$w: E \to \mathbb{R}$ 为边权重函数。它独立于主题DAG，用于捕捉**非层次关系**（因果、共现、时序关联等）。

#### 3.1 边的来源与生成

**来源一：记忆共现边（共现关联）**

由于每条记忆标注了多个主题，同一条记忆的不同主题之间天然形成共现关系。

对每对主题 $(t_i, t_j)$，定义共现频次：

$$cooccur(t_i, t_j) = |M_i \cap M_j|$$

使用**归一化逐点互信息（NPMI）** 度量关联强度，消除频率偏差：

$$PMI(t_i, t_j) = \log\frac{P(t_i, t_j)}{P(t_i) \cdot P(t_j)}$$

$$NPMI(t_i, t_j) = \frac{PMI(t_i, t_j)}{-\log P(t_i, t_j)}$$

其中概率估计为：

$$P(t_i) = \frac{|M_i|}{|M_{total}|}, \quad P(t_i, t_j) = \frac{|M_i \cap M_j|}{|M_{total}|}$$

NPMI 取值范围为 $[-1, 1]$，+1表示完全共现，0表示独立，-1表示互斥。

**建边条件**：$NPMI(t_i, t_j) > \tau_{co}$（推荐 $\tau_{co} = 0.1$）且 $cooccur(t_i, t_j) \geq n_{min}$（推荐 $n_{min}=2$，避免单次共现产生噪声边）

**来源二：LLM推断的因果/语义边**

利用LLM判断主题标签之间的因果或强相关性（比如”工作压力”→”失眠”），适用于语义直觉上相关但共现数据不足的场景。

对每对候选主题提交LLM，prompt要求判断：
- 是否存在关联？（yes/no）
- 关联类型：因果（causal）、条件（conditional）、互补（complementary）
- 关联强度：一个 $[0,1]$ 的评分 $s_{llm}$
- 方向：$t_i \to t_j$ 或 $t_j \to t_i$ 或双向

> 为减少LLM调用量，仅对”来源一”未建立边但向量相似度处于中等区间（$0.3 < cos(\mathbf{e}_i, \mathbf{e}_j) < 0.6$）的主题对进行LLM判断——相似度太高可能已被来源一覆盖或属于同一DAG分支，太低则大概率无关联。

**来源三：时序共现边（可选）**

当两个主题频繁在相近的时间窗口（比如同一次对话、或相隔不超过k轮的不同对话）内出现，这种时间上的共现也暗示关联：

$$temporal\_cooccur(t_i, t_j) = \sum_{s \in Sessions} \mathbb{1}[t_i \in s \wedge t_j \in s]$$

可使用与来源一相同的NPMI框架，但在session级别（而非memory级别）计算

#### 3.2 边权重的统一计算

对于可能由多个来源同时产生的边，统一权重计算公式：

$$w(t_i, t_j) = w_1 \cdot NPMI_{mem}(t_i, t_j) + w_2 \cdot s_{llm}(t_i, t_j) + w_3 \cdot NPMI_{temp}(t_i, t_j)$$

其中：
- $w_1, w_2, w_3$ 为各来源的可信度权重，推荐 $w_1 = 0.5, w_2 = 0.3, w_3 = 0.2$
- 若某个来源没有产生边，则该项为0
- 最终归一化到 $[0,1]$

**边方向的确定**：对于共现边，使用条件概率的不对称性来确定方向：

$$direction(t_i \to t_j) = P(t_j | t_i) - P(t_i | t_j) = \frac{|M_i \cap M_j|}{|M_i|} - \frac{|M_i \cap M_j|}{|M_j|}$$

若 $direction > \epsilon$，则为 $t_i \to t_j$（表示”从 $t_i$ 更容易联想到 $t_j$”）；若 $|direction| \leq \epsilon$，则视为双向边。对于LLM来源的边，直接使用LLM给出的方向。

#### 3.3 图的剪枝与稀疏化

避免关联图过于稠密导致检索扩展时噪声过多：

1. **权重阈值剪枝**：移除 $w(t_i, t_j) < \tau_w$ 的边（推荐 $\tau_w = 0.15$）
2. **度数约束**：每个节点最多保留top-K条出边（推荐K=5），按权重排序保留
3. **统计显著性检验**（可选）：使用对数似然比检验（Log-Likelihood Ratio），仅保留统计显著的共现边（$p < 0.05$）

#### 3.4 图的动态更新策略

**增量更新（每次新记忆写入时）：**
1. 新记忆涉及主题集合 $T_{new} = \{t_a, t_b, ...\}$
2. 对 $T_{new}$ 中的每对主题，更新共现计数 $cooccur(t_i, t_j) += 1$
3. 重新计算涉及到的边的NPMI值（增量更新，无需全局重算）
4. 若重算后超过/低于阈值，添加/移除边
5. **对已有边应用衰减-强化更新**：将历史衰减和新证据叠加统一处理，避免权重不连续跳变：

$$w_{new} = \underbrace{w_{old} \cdot e^{-\frac{\ln 2}{t_{half}} \cdot \Delta t}}_{\text{先衰减}} + \underbrace{\eta \cdot \left(1 - \frac{w_{decayed}}{w_{max}}\right) \cdot \Delta w}_{\text{再强化}}$$

其中：
- $w_{old}$ 为上次存储的权重，$\Delta t$ 为距上次更新的时间间隔
- $\eta = 0.2$ 为学习率
- $w_{max} = 1.0$ 为权重上限。$(1 - w_{decayed}/w_{max})$ 项实现**边际递减**：已很强的关联边增长缓慢，弱边增长较快，防止头部边权重失控
- $\Delta w$ 为本次新增证据的信号强度，即3.2节统一权重公式的增量值

> **为什么不直接用NPMI重算替代权重？** 纯NPMI重算会导致"一条长期未活跃的边因为共现基数没变而维持原权重"——它无法反映时间衰减。衰减-强化公式则将时间遗忘和频率积累统一到一次更新中，更符合记忆的自然规律（参考ACT-R认知架构中base-level activation公式的设计思想：$B_i = \ln(\sum_j t_j^{-d})$，同时编码频率和新近度）。

$$NPMI_{updated}(t_i, t_j) = \frac{\log\frac{|M_i \cap M_j|_{new} \cdot |M_{total}|_{new}}{|M_i|_{new} \cdot |M_j|_{new}}}{-\log\frac{|M_i \cap M_j|_{new}}{|M_{total}|_{new}}}$$

**定期重构（主题数量变化超过阈值时，或每N条记忆后）：**
1. 全局重算所有NPMI
2. 重新触发LLM因果判断（仅对新出现的主题对，已有LLM判断结果缓存复用）
3. 重新执行剪枝策略
4. 检测并处理孤立节点（无任何关联边的主题）

**边的衰减（可选）：** 引入时间衰减因子，使关联图反映近期记忆的主题关系：

$$w_{decayed}(t_i, t_j) = w(t_i, t_j) \cdot e^{-\frac{\ln 2}{t_{half}} \cdot \Delta t}$$

其中 $\Delta t = T_{now} - T_{last\_update}$，$T_{last\_update}$ 为该边最后一次因新记忆而更新的时间，$t_{half}$ 为**半衰期**（即边权重衰减到一半所需的时间）。推荐 $t_{half} = 30$ 天。

> **为什么用半衰期而非衰减率**：半衰期的物理含义更直观——"30天无强化则权重减半"比"$\lambda=0.977$/day"更易理解和调参。两者数学上完全等价：$\lambda = e^{-\ln 2 / t_{half}}$，当 $t_{half}=30$ 天时 $\lambda \approx 0.977$/day。注意原推荐的 $\lambda=0.95$/day 对应半衰期仅约13.5天，对于个人长期记忆场景衰减偏快。
>
> 衰减采用**惰性计算**：不需要后台定时刷新所有边权重，仅在查询访问或剪枝维护时按需计算 $w_{decayed}$，存储 $(w, T_{last\_update})$ 即可。

#### 3.6 基于Personalized PageRank的跨主题扩展算法

当检索需要跨主题扩展时（详见下节"主题路由的记忆检索"），在主题关联图上使用**Personalized PageRank（PPR）** 进行spreading activation，替代简单的1-hop邻居扩展，实现多跳关联推理。

> **参考**：HippoRAG（Gutierrez et al., 2023）将PPR应用于知识图谱检索，模拟人类海马体的联想记忆模式——从激活点出发，沿关联路径逐步扩散，自然实现多跳推理。

**PPR迭代公式：**

$$\mathbf{p}^{(k+1)} = \alpha \cdot M \cdot \mathbf{p}^{(k)} + (1-\alpha) \cdot \mathbf{s}$$

其中：
- $\mathbf{s} \in \mathbb{R}^{|V|}$：种子向量（个性化向量）。由query直接命中的主题构成，若命中主题集合为 $T_q = \{t_{q_1}, ..., t_{q_k}\}$，则 $s_i = \frac{1}{k}$ 当 $t_i \in T_q$，否则 $s_i = 0$
- $M \in \mathbb{R}^{|V| \times |V|}$：关联图的**列归一化加权邻接矩阵**，$M_{ij} = \frac{w_{decayed}(t_j, t_i)}{\sum_l w_{decayed}(t_j, t_l)}$，即从 $t_j$ 出发到 $t_i$ 的转移概率
- $\alpha = 0.85$：阻尼系数。$\alpha$ 越大，扩散越远（更多多跳结果）；$\alpha$ 越小，越集中在种子节点附近
- 迭代 $K$ 轮至收敛（通常5-10轮即可，或当 $\|\mathbf{p}^{(k+1)} - \mathbf{p}^{(k)}\|_1 < \epsilon$ 时终止，推荐 $\epsilon = 10^{-6}$）

**也可使用闭式解**（当图规模较小时，直接矩阵求逆更高效）：

$$\mathbf{p} = (1-\alpha)(I - \alpha M)^{-1} \mathbf{s}$$

**扩展结果**：取激活值 $\mathbf{p}$ 中排名top-K的主题（排除已在种子集中的），作为跨主题扩展的候选。推荐 $K=3 \sim 5$。

**示例**：查询"我最近为什么效率低" → 种子主题 $T_q = \{\text{工作效率}\}$ → PPR扩散后激活排名：工作压力(0.31) > 失眠(0.18) > 加班(0.15) > ... → 扩展到"工作压力"和"失眠"主题，获取其下记忆参与最终排序

## 主题路由的记忆检索

**总体思路总结：**针对用户查询短小模糊、隐含主题不明确且常伴随跨主题推理需求的问题，本课题拟提出一种主题路由驱动的记忆检索框架，将主题识别与主题约束前置到记忆召回过程。该框架首先综合主题标签、主题摘要和关键词等信号识别候选主题，再借助主题DAG完成主题集合的扩展与收缩，并在主题内执行细粒度记忆排序；当查询包含因果追问或当前结果不足时，再利用主题关联图进行跨主题扩展。通过“先主题路由、后主题内检索、再跨主题扩展”的分层策略，该方法能够有效降低噪声记忆干扰，提升主题过滤后的检索精度、召回率与复杂问答场景下的解释能力。

查询->目标主题->跨主题扩展->主题内记忆

**具体方法：**

​	1.判断query的相关主题，可考虑使用的方法：向量相似度（同时比较标签嵌入和摘要嵌入）、关键词匹配等，或多方案查找并计算主题的匹配得分

​	2.利用主题DAG调整候选主题集：如果主题数量不足，沿DAG向上扩展，将同一父节点下的兄弟主题加入；如果匹配到的主题数量过多，则利用DAG进行缩减操作——删掉有子主题存在的父主题。例如匹配到”技术学习”、”Python”和”健康”，主题数量上限规定为2，则根据DAG删除”技术学习”（因为子主题”Python”已存在），保留”Python”和”健康”。注意DAG中一个主题可能有多个父节点，缩减时需避免误删。

​	3.在候选主题下的记忆集挑选合适的memory，挑选方法可考虑（向量相似度、关键词匹配等）

​	4.利用主题关联图跨主题扩展：使用3.6节中的PPR算法，以当前候选主题为种子节点，在关联图上进行spreading activation，取top-K扩展主题及其下的记忆。例如问题”我为什么最近失眠”，种子主题为”睡眠”，PPR扩散后扩展到”工作压力”主题，获得该主题下的记忆”连续加班三天”

**何时进行跨主题扩展：**

1.所有情况都进行，可以考虑给跨主题的memory缩小权重，更加保留不跨主题的高质量memory

2.出现推理关键词比如“为什么”“什么原因”等等

3.当前检索到的memory的相关度得分都过低，进行扩展
